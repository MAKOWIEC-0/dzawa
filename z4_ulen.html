2)dlaczego jest to język skryptowy
JavaScript jest językiem skryptowym, ponieważ nie wymaga kompilacji przed uruchomieniem. Jego kod jest interpretowany i wykonywany w czasie rzeczywistym przez przeglądarkę lub środowisko wykonawcze (np. Node.js).
3)gdzie jest wykonywany JS (klient lub serwer)
JavaScript jest wykonywany po stronie klienta (w przeglądarce), ale dzięki środowiskom takim jak Node.js, może być również wykonywany po stronie serwera.
4)dlaczego jest JS to bezpieczne narzędzie
JavaScript jest bezpieczny, ponieważ działa w izolowanym środowisku (tzw. sandbox) w przeglądarce, co uniemożliwia bezpośredni dostęp do systemu operacyjnego użytkownika. Ponadto przeglądarki implementują mechanizmy zabezpieczeń, takie jak CORS czy Same-Origin Policy.
5)czy jest to język obiektowy , jeśli tak to dlaczego.
Tak, JavaScript jest językiem obiektowym, ponieważ obsługuje koncepcje programowania obiektowego, takie jak klasy (od ES6), obiekty, dziedziczenie, polimorfizm i hermetyzacja.
6)wymień zalety JS:
Zaleta1 Łatwość nauki i wszechstronność.
Zaleta2 Możliwość działania zarówno po stronie klienta, jak i serwera.
Zaleta3 Bogata ekosystem bibliotek i frameworków (np. React, Angular, Vue).
7)Framework to:
Framework to gotowa struktura oprogramowania, która dostarcza narzędzi, bibliotek i wzorców projektowych, aby ułatwić tworzenie aplikacji.
8)Do aplikacji mobilnych można użyć Framework
Do tworzenia aplikacji mobilnych można użyć frameworków takich jak React Native, Ionic lub Flutter.
9)Aplikacje responsywne to:
Aplikacje responsywne to aplikacje, których interfejs użytkownika dostosowuje się do różnych rozmiarów ekranów, zapewniając optymalne wrażenia użytkownika na różnych urządzeniach (np. komputerach, tabletach, smartfonach).

11)Przepisz linie kody pod nimi wytłumaczenie jak działa ta ten fragment kodu
window.open("obraz.html","okienko","toolbar=no,directories=no,menubar=no,height=280,width=160,top=200,left=200");
Ta linia kodu otwiera nowe okno przeglądarki z adresem “obraz.html”. Okno ma określone wymiary (280x160 pikseli), jest umieszczone na pozycji (200, 200) na ekranie oraz nie zawiera paska narzędzi, katalogów ani menu.
